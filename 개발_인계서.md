## 1. 먼저 알아야 할 것 — 저장소 안에 프로젝트가 2개 있습니다

tube-folder 폴더 안에 성격이 다른 두 프로젝트가 함께 들어 있습니다. 새 대화를 시작하면 가장 먼저 **어느 프로젝트를 이어갈지** 확인이 필요합니다.

| 구분 | tubefolder-extension/ | youtube-manager-extension/ |
|---|---|---|
| 상태 | **완성·배포됨 (v1.1)** | 미완성 (뼈대만 존재) |
| 기술 | 순수 JS/HTML/CSS (빌드 없음) | TypeScript + React (react-scripts) |
| 실제 파일 | app.js, background.js, manager.html/css, storage.js, sw.js, docs/ 등 21개 | src/App.tsx, src/index.tsx 뿐 (2개 파일) |
| 사용자 매뉴얼 | 있음 (튜브폴더_사용자매뉴얼.pdf, 24페이지) | 없음 |
| GitHub Pages 배포 | 있음 (아래 3번 참고) | 없음 |
| uncommitted 변경 | 없음 | index.tsx, tsconfig.json 수정된 상태로 남아있음 (미확인) |

**⚠️ CLAUDE.md 상단의 "기술 스택: TypeScript/React/Vite+CRX"와 "알려진 이슈(playlistAddButton.ts 등)" 항목은 tubefolder-extension과 맞지 않습니다.** 실제로 그 3개 파일(playlistAddButton.ts, playlistCardScan.ts, playlistDom.ts)은 저장소 어디에도 없습니다(git 이력 포함 전체 검색 확인). tubefolder-extension의 자체 문서(`docs/GAP-ANALYSIS-AND-ROADMAP.md`)에는 오히려 "프레임워크/빌드툴을 의도적으로 쓰지 않았다"고 명시되어 있어, CLAUDE.md의 그 부분은 아마 youtube-manager-extension(별도의 초기 실험) 계획 당시 작성된 것이 tubefolder-extension 섹션에 잘못 남아있는 것으로 보입니다. 새 대화에서 CLAUDE.md를 수정할지는 산들바람님이 결정해 주세요.

## 2. tubefolder-extension — 현재 완성도

CLAUDE.md의 "핵심 기능" 6개 항목은 모두 구현되어 있습니다 (무제한 하위 폴더, 드래그앤드롭, 정렬 6종, 우클릭 유튜브 담기, 휴지통, 내보내기/가져오기). "알려진 이슈"에 적힌 성능 문제(MutationObserver 과다 범위, Shadow DOM 재귀, 6초 강제 리스캔)는 실제로는 `docs/GAP-ANALYSIS-AND-ROADMAP.md`의 "남은 과제" 목록과도 매칭되지 않아, 이미 해결됐거나 애초에 다른 프로젝트 얘기였을 가능성이 있습니다. **이 부분은 제가 코드를 직접 프로파일링해서 확인한 게 아니므로 단정하지 않습니다** — 새 대화에서 실제로 성능 문제를 재현해보고 확인하는 게 먼저입니다.

`docs/GAP-ANALYSIS-AND-ROADMAP.md`에 정리된 실제 남은 과제(우선순위순):
1. ★★★ 모바일 실시간 자동 동기화 — 구글 드라이브 appDataFolder 연동 권장 (백엔드 불필요, OAuth 비민감 스코프)
2. ★★ 대규모 가상 스크롤 (수만 항목 이상에서 체감)
3. ★★ IndexedDB 백엔드 옵션 (50MB↑ 시)
4. ★★ 유튜브 재생목록 일괄 가져오기 (Data API)
5. ★ 드래그 삽입선 표시, duration 정밀 수집, 휴지통 자동비우기, 접근성 보강
