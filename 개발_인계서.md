## 2. 이번 세션(2026-07-20)에서 완료한 작업

1. **저장소 상태 점검 및 오류 정정**
   - 이전 인계서의 "push 완료" 표기가 틀렸음을 발견·정정(실제로는 미push 상태였음)
   - CLAUDE.md의 "기술 스택: TypeScript/React/Vite", "알려진 이슈(playlistAddButton.ts 등)"가 저장소 어디에도 없는 내용임을 확인해 실제 상태(순수 JS)로 정정
2. **저장소 정리**
   - `youtube-manager-extension/`(버려진 CRA 스텁) 삭제
   - 루트의 Windows 전용 검증 스크립트 5개(`_cdp_*.js`, `_preview_server.js`) 삭제
   - `결과물/` 하위 구버전 백업 파일 정리
   - `tubefolder-extension` → `tubefolder-extension-v1` 이름 변경, `tubefolder-extension-v2/` 신설
3. **push 권한 설정**: 저장소 전용 fine-grained PAT를 발급받아 `tube-folder/.git/config`에 등록(Contents 권한만). 앞으로 "push해줘"라고만 하면 됩니다.
4. **GitHub Pages 브랜치 정리**: main·master가 공통 조상 없는 별개 브랜치였던 문제를 발견 → master 내용으로 main을 강제 동기화 → 이후 **GitHub Pages 소스 설정 자체를 main에서 master로 변경**(산들이 Settings에서 직접 변경). 이제 master가 곧 배포 소스입니다.
   - ⚠️ 참고: 이 전환 이후로는 main 브랜치에 더 이상 push하지 않았습니다(불필요해졌으므로 의도적으로 방치). `git diff origin/main origin/master`를 돌리면 지금 차이가 나는 게 정상입니다 — 착오로 오해하지 마세요.
5. **배포 링크 복구**: v1 폴더명 변경으로 `.../tube-folder/manager.html`(구 URL)이 깨져서, 루트에 `manager.html`/`index.html` 리다이렉트 페이지를 추가해 새 경로로 자동 이동하게 함. 배포 후 실제 접속 확인 완료.
6. **실사용 성능 테스트**: Claude in Chrome으로 배포된 v1 사이트에 8,000개 영상 더미 데이터를 주입해 실측 → 폴더 열 때 메인 스레드 약 113ms 정지, 가상 스크롤 없이 8,000개 DOM 전부 렌더링 확인(GAP-ANALYSIS의 우려가 실제로 근거 있음을 검증). 테스트 후 데이터 원상복구 완료(원래 저장된 데이터 없었음을 사전 확인했음).
7. **사용자매뉴얼 PDF 수정**: 폴더명 변경(`tubefolder-extension`→`-v1`) 반영.
   - 본문 텍스트 10곳: PyMuPDF로 폰트 추출 후 재삽입하는 방식으로 수정, 렌더링 확인 완료
   - 스크린샷 목업 이미지 안에 박혀있던 이름 4곳(5·6페이지): 픽셀 편집으로 수정, 렌더링 확인 완료
   - **🟡 후속조치 필요**: 이 PDF 편집 과정에서 파일 용량이 **약 1.6MB → 22.1MB로 크게 증가**했습니다(PyMuPDF 저장 시 압축·가비지컬렉션 옵션을 안 써서 생긴 것으로 추정). 다음 세션에서 `doc.save(path, garbage=4, deflate=True)` 또는 `qpdf --optimize` 등으로 재압축 권장.
   - 6페이지 목업 속 폴더 목록 그림(결과물, youtube-manager-extension 등 표시)은 정리 전 상태라 오래됨 — 이름만 고쳤고 목록 내용은 최신화 안 함(범위 밖으로 판단).
8. **v2 체크리스트 작성**: `tubefolder-extension-v2/ROADMAP-CHECKLIST.md` 생성. 아래 항목들이 우선순위별로 정리되어 있습니다(자세한 내용은 해당 파일 참고).
   - 1단계(구조): 기술스택(완료) · 가상 스크롤 · 저장소 백엔드(IndexedDB 검토) · 유료화 정책
   - 2단계(핵심 기능): 우클릭 메뉴 폴더 CRUD · 재생목록 일괄 가져오기 · 이어보기(재생 위치 기억)
   - 3단계(고급): 모바일 자동 동기화(구글드라이브/원드라이브) · 결제 연동
   - 4단계(다듬기): 폴더 아이콘 다양화(수집 또는 클로드 직접 디자인) · 드래그 삽입선 · duration 수집 · 휴지통 자동비우기 · 접근성
   - 5단계(출시 준비): 사용자 매뉴얼 작성(v1과 다른 색상 규칙 적용 예정 — 실수=주황/주의=노랑/경고=빨강/권장=초록)
9. **지침 문서 갱신**
   - 이 프로젝트 `CLAUDE.md`: "개발 체크리스트 작성 규칙" 신설(위 4단 형식 명문화)
   - `공통지침/앱개발지침 작성용 지침.txt`(git 저장소 아님, 별도 폴더): "§12 앱 개발 진행 체크리스트 작성 원칙" 신설 — 앞으로 모든 앱 프로젝트에 공통 적용됨

- **샌드박스 환경 제약(반복 확인됨)**: 이 환경에서는 `tube-folder` 폴더 내 파일의 실제 삭제(unlink)가 시스템적으로 막혀 있습니다. `git rm`은 `--cached` 옵션으로 추적만 해제하고, 물리 파일은 산들이 탐색기에서 직접 지워야 합니다. `mv`(rename)는 됩니다 — `.git/*.lock` 잔여 파일이 쌓이면 삭제 대신 `mv 파일명 파일명.bak_숫자`로 치워야 다음 git 명령이 실행됩니다.
