## 2. 이번 세션(2026-07-20~21)에서 완료한 작업

1. **사용자매뉴얼 PDF 재압축**: 지난 세션에서 22.1MB로 불어났던 파일을 2.07MB로 축소(원인: PyMuPDF 저장 시 가비지컬렉션 옵션 미사용으로 편집 전 이미지 원본이 orphan 객체로 남아있었음 → `garbage=4, deflate=True, clean=True`로 재저장). 24페이지 전체 텍스트 diff 0건·qpdf 구조검사 통과·5·6페이지 렌더링 육안 확인으로 검증. 커밋 `fccd6ad`, push 완료.

2. **v2 1단계(구조 설계) 4/4 전부 확정**:
   - 가상 스크롤: `@tanstack/react-virtual`(헤드리스) + `@dnd-kit/sortable`. v1의 마퀴선택·키보드이동·드래그삽입선을 그대로 재현하려면 제어권이 큰 헤드리스 방식이 필요해 `react-virtuoso`보다 이쪽 채택.
   - 저장소 백엔드: `chrome.storage.local` + `unlimitedStorage` 유지(IndexedDB 전환 안 함) — 8,000개 실측에서 병목은 저장소가 아니라 DOM 렌더링이었음이 근거.
   - 유료화 정책: 시장 조사(Tab Folio·Toby·Leap·Bookmarker·Raindrop.io·Tab Group Vault 등) 근거로 제안 후 산들이 확정 — **무료**: 핵심 정리 기능 전부 + 이어보기. **유료**(1회 결제 ₩15,000~20,000): 폴더/영상 개수 제한 해제, 클라우드 동기화, 대량 재생목록 가져오기.

3. **v2 2단계(핵심 기능) 진행 — Claude Code에서 실제 코딩**:
   - v2 프로젝트 최초 스캐폴딩(Vite+React+TS, MV3 3개 빌드 진입점)과 함께 "우클릭 메뉴 폴더 CRUD"를 유튜브 페이지 위 Shadow DOM 미니 팝업으로 구현(매니저 탭 전환 없이 처리). storage.js를 TS로 이식, 개발 중 버그 2건 발견·수정. 커밋 `9162447`.
   - "유튜브 재생목록 일괄 가져오기" 구현 — 착수 전 확인 결과 oEmbed는 재생목록 나열 기능 자체가 없어서, 재생목록 페이지에 내장된 `ytInitialData`를 파싱하는 방식(API 키 불필요)으로 변경해 구현. 100개 초과 시 `youtubei/v1/browse` 이어받기 + 진행상황 실시간 표시. 커밋 `8e76213`.
   - 매뉴얼 아이콘 모양 변경(색상 규칙은 유지, 모양만): 자주하는실수 ✕→🚫, 주의사항 ℹ️→📌, 경고 ⚠️→🚨. 커밋 `bf38ecd`.
   - **🟡 후속조치 필요**: 두 기능(우클릭 CRUD, 재생목록 가져오기) 모두 개발 환경 제약으로 타입체크·모킹 테스트까지만 검증했고, **실제 Chrome에 압축해제 확장으로 로드한 육안 확인은 아직 안 됨**(`tubefolder-extension-v2/README.md`의 "확장으로 로드해서 확인하기" 참고). 특히 재생목록 100개 초과 대형 재생목록의 실제 이어받기 동작은 실제 URL로 확인 필요.

4. **Claude Code 데스크톱 앱 사용 관련 확인**: 세션이 다른 프로젝트(mindmap-app) 태그로 묶여 있어도 절대경로를 지정하면 tube-folder에 정확히 저장됨을 git 커밋으로 직접 검증함(파일 소재는 세션 태그가 아니라 실제 커밋 경로로 확인해야 함). 다만 혼동 방지를 위해 새 세션을 열 때는 프로젝트 폴더를 tube-folder로 명시 지정하는 걸 권장(최근 목록에 없으면 "폴더 열기"로 직접 탐색).

5. **인계서 작성 방침**: 원칙적으로는 CLAUDE.md·ROADMAP-CHECKLIST.md가 항상 최신으로 갱신되고 있어 별도 인계서 없이 새 대화방을 열어도 됩니다. 이번 인계서는 산들의 직접 요청으로 작성했습니다.

## 3. Git / GitHub 현재 상태 (확인된 사실)

- 원격 저장소: `https://github.com/fics01287-arch/tube-folder.git` (Public)
- 로컬 `master` = origin/master, 최신 커밋 `bf38ecd`. **완전히 동기화된 상태**(ahead/behind 0).
- GitHub Pages 소스: master 브랜치(변경 없음, 이전 인계서 참고). main은 계속 의도적으로 방치 중.
- **샌드박스 환경 제약(이번 세션에도 반복 확인됨)**: `.git/index.lock`·`HEAD.lock`·`refs/remotes/origin/*.lock`·`objects/*/tmp_obj_*` 같은 잔여 lock 파일이 커밋·push 도중 자주 남습니다(실제 실행 중인 git 프로세스는 없음 — 아마 산들 컴퓨터의 Claude Code 세션이 같은 폴더를 동시에 들여다봐서 생기는 것으로 추정). 삭제(unlink)는 막혀 있으니 `mv 파일명 파일명.bak_숫자`로 치우고 재시도하면 됩니다 — 이번 세션에서도 이 방법으로 매번 해결됨.
