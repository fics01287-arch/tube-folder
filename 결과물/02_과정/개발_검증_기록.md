## 5. 실제 Chrome 로드 검증 (CDP) — 2026-06-23

`_cdp_test.js`로 **실제 설치된 Chrome 149**에 확장을 로드해 검증(임시 프로필, 사용자 프로필 미접촉).

### 관측 결과 (원본 OneDrive 한글 경로에서 직접 로드, `TF_SRC=1`)
```
확장 로드됨 (Extensions.loadUnpacked) → SW 타깃 발견 → manager.html 열림
test_initial   : {title:"튜브폴더", items:["휴지통"], hasChromeStorage:true}
test_created   : ["새 폴더","새 폴더 (2)","휴지통"]
test_storage   : {saved:true, nodeCount:4, version:1}     ← chrome.storage.local 실제 저장
test_afterReload: ["새 폴더","새 폴더 (2)","휴지통"]       ← 리로드 후 영속 (요구 7)
```
→ 실제 확장 컨텍스트에서 **chrome.storage API 존재·렌더·CRUD·중복이름·휴지통고정·영속** 전부 확인. ASCII 임시 경로 로드도 동일 통과.

### ⚠️ 중요한 함정 (Chrome 137+/149)
- **자동화의 `--load-extension` 커맨드라인 스위치는 보안상 무시됨**(`chrome://extensions` 카드 0개로 확인). 자동 로드는 **CDP `Extensions.loadUnpacked` + `--enable-unsafe-extension-debugging`** 로 해야 한다.
- ✅ **이 제약은 자동화 한정.** 사용자가 `chrome://extensions`에서 **수동 "압축해제된 확장 로드"** 하는 정상 설치 경로는 **영향 없음** — 정상 동작.
- 자동화 시 한글/OneDrive 경로는 ASCII 임시 폴더 복사 권장(`Extensions.loadUnpacked`는 path 인자라 한글도 되지만, OneDrive 온라인전용 플레이스홀더 회피).

### 재현
```
# ASCII 임시 복사본으로 로드(기본, 견고)
node _cdp_test.js
# 원본 OneDrive 한글 경로에서 직접 로드
TF_SRC=1 node _cdp_test.js   # (PowerShell: $env:TF_SRC="1"; node _cdp_test.js)
```
