### 🤖 수행 (`_cdp_test.js`, Node 내장 WebSocket으로 CDP 구동)
- 환경 확인: Node v24.15.0, Chrome 149(`C:\Program Files\Google\Chrome\Application\chrome.exe`). 임시 프로필 사용(본인 프로필 미접촉).
- 1차 시도 `--load-extension`: ❌ ERR_FILE_NOT_FOUND/카드 0개 → 디버깅.
  - 매처를 background.js로 한정 → 우리 SW 안 보임. ASCII 임시경로 복사 → 여전히 실패. `DisableLoadExtensionCommandLineSwitch` 비활성화 → 실패. `chrome://extensions` 캡처 → **카드 0개**.
  - **결론: Chrome 137+/149는 보안상 `--load-extension` 스위치를 무시.**
- 해결: CDP **`Extensions.loadUnpacked` + `--enable-unsafe-extension-debugging`** 로 전환 → 성공.
- 실측(원본 OneDrive 한글 경로 직접 로드):
  - `test_initial {title:"튜브폴더", items:["휴지통"], hasChromeStorage:true}`
  - `test_created ["새 폴더","새 폴더 (2)","휴지통"]`
  - `test_storage {saved:true, nodeCount:4, version:1}` ← chrome.storage.local 실제 저장
  - `test_afterReload ["새 폴더","새 폴더 (2)","휴지통"]` ← **리로드 후 영속(요구7) 실증**
- 스크린샷 전달. `결과물/02_과정/개발_검증_기록.md §5` 기록, `_cdp_test.js`를 결과물에 포함.
