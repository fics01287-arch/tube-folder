# 진화 타임라인 (Process & Evolution)

튜브폴더가 만들어진 전 과정과 **무엇을·왜** 바꿨는지의 기록. (자가진화의 출발점 자료)

---

## Phase 0 — 출발점 진단
- 기존 `youtube-manager-extension/`을 스캔 → **빈 Create-React-App(CRA) 스텁**이었음.
  - `src/App.tsx` = `"YouTube 매니저 로딩 완료"` 한 줄. manifest 없음, 폴더/정렬/휴지통 로직 **0%**.
- **핵심 판단:** CRA(`react-scripts`)는 **MV3 확장을 만들 수 없다**(인라인 스크립트가 확장 CSP 위반).
- **결정:** CRA를 고쳐 쓰지 않고 **의존성 0의 순수 MV3 확장으로 신규 작성**. 기존 스텁은 삭제하지 않고 보존.

## Phase 1 — v1.0 (탐색기형 확장 골격)
요구 1~7·9를 구현:
- 정규화 트리(노드 맵 + `parentId`) → 무제한 중첩
- CRUD(생성/이동/복사/삭제) · 정렬(이름/날짜/유형/크기/없음 × 오름·내림)
- 휴지통 루트 최하단 고정 · 6종 보기모드 · `chrome.storage.local` 영속 · 단일 zip 배포
- 컨텍스트 메뉴 "튜브폴더에 추가" · 드래그&드롭 · 인라인 이름변경
- **검증:** node --check 통과, 미리보기에서 폴더 생성·휴지통 고정·콘솔 에러 0 확인.

## Phase 4 — 하이브리드 + 에이전트 문서 (최종)
- **플랫폼 결론:** 크롬 확장 단독은 #8(모바일) 불가 → **데스크톱=확장 / 모바일=PWA** 하이브리드.
- **PWA화:** `manager.webmanifest` + `sw.js`(네트워크 우선 캐시) + 아이콘 192/512 추가. 같은 코어가 두 런타임에서 동작.
- **자가진화 지침:** `AGENTS.md` 작성(실행·검증 절차, 코드지도, 불변식 I1~I9, 회귀 R1~R4, 함정 P1~P8, 진화 루프).
- **검증:** PWA 추가 후에도 렌더 정상·매니페스트 링크 확인·콘솔 에러 0.

## 설계 의사결정(되짚기용)
1. **프레임워크 미사용** — 단일 zip 배포 + CSP 안전 + 모바일/확장 코어 1벌.
2. **영상=참조 저장** — 유튜브 재생목록은 평면 구조라 중첩은 자체 가상 계층. 노드당 ~200B로 경량.
3. **storage.local 우선, IndexedDB는 옵션** — onChanged 교차탭 동기화를 무료로 얻음. 초대형 시 교체.
4. **수동정렬은 키보드(Alt+↑/↓)** — 드래그 히트테스트 모호성 회피, 결정적 동작.
